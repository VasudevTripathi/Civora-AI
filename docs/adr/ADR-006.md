# ADR-006: LLM Provider — Google Gemini via Google GenAI SDK

## Decision
We will use **Google Gemini** models via the official `google-genai` Python SDK as our primary AI reasoning engine.

## Context
Civora AI needs an LLM with high analytical capability, support for large context windows (to process long legal acts), structured output capabilities (to parse responses into strict Pydantic schemas), and fast execution speed.

## Alternatives Considered
* **OpenAI GPT Models**: Excellent capabilities, but has higher pricing tiers for high token throughput and doesn't offer the native 1-million+ token context window built into Gemini 1.5 Pro, which is crucial for ingestion of complete legal publications.
* **Self-hosted Open-Source Models (e.g. Llama 3)**: Provides complete privacy control but introduces massive infrastructure costs, complex scaling challenges, and higher operational latency for startup phase.

## Consequences
* **Positive**:
  * Massive context window capabilities allow complete acts or books to be read as reference context.
  * Robust structured JSON outputs that map cleanly to Pydantic models.
  * Cost-effective pricing tiers for enterprise scalability.
* **Negative**:
  * Tightly couples our AI service adapter to the Google GenAI API syntax. We must encapsulate this adapter strictly behind an interface to allow model portability.
