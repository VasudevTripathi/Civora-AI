# ADR-005: AI System Design — Retrieval-Augmented Generation (RAG)

## Decision
We will employ **Retrieval-Augmented Generation (RAG)** combined with Qdrant Vector DB to answer compliance questions.

## Context
Government regulations require exact factual compliance. An AI system that hallucinates due dates, state filing fees, or form names can cause severe legal penalties for users. Large language models (LLMs) have broad general knowledge but cannot guarantee factual precision for specific, changing municipal guidelines unless grounded with direct reference contexts.

## Alternatives Considered
* **Model Fine-Tuning**: Fine-tuning an LLM on government documents is expensive, time-consuming, and does not prevent hallucination. It also cannot be updated in real-time when a state changes a fee or deadline.
* **Basic LLM Prompting**: Passing all compliance textbooks inside a single system prompt consumes too many tokens, is expensive, and degrades query accuracy due to context window noise.

## Consequences
* **Positive**:
  * Guarantees high accuracy by providing the exact statutory clause context to the generator.
  * Allows real-time updates: updating a PDF source instantly updates the RAG retrieval database without model training.
  * Provides explainability: every AI response can reference the source PDF page or legal act clause.
* **Negative**:
  * Requires building and managing a robust parsing, chunking, and embedding pipeline for unstructured PDF documents.
